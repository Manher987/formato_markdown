Markdown: Historia, utilidad y casos de uso

1) ¿Qué es Markdown?

(Explicación con sus propias palabras)

Es un lenguaje de marcado (por lo que investigué, es un sistema que usa etiquetas para escribir y estructurar el contenido de un documento) que permite valga la redundancia, dar formato a texto usando únicamente carácteres de teclado. Es de texto plano.

2) Origen e historia de Markdown

  • ¿Quién lo creó?
    Fue desarrollado por John Gruber (creador del sitio Daring Fireball), con una colaboración fundamental de Aaron Swartz en la parte de la sintaxis y el diseño del lenguaje.
  • ¿En qué año?
    Fue lanzado originalmente en 2004.
  • ¿Con qué propósito?
    Su objetivo era permitir que las personas escribieran usando un formato de texto plano fácil de leer y de escribir, que pudiera convertirse estructuralmente en HTML válido. Gruber quería que el texto fuente fuera legible incluso si no se procesaba, inspirándose en las convenciones existentes para dar formato a correos electrónicos en texto plano.

3) ¿Para qué sirve Markdown?

  1. Escritura ágil de contenido web: Permite generar código HTML complejo (como tablas, listas y enlaces) usando solo caracteres simples. Esto ahorra tiempo y evita errores de sintaxis comunes en HTML (como olvidar cerrar una etiqueta </div>).
  2. Estandarización de documentación: Se utiliza para crear manuales y guías técnicas que se mantienen legibles tanto en un editor de código como en un navegador, facilitando la colaboración entre desarrolladores.
  3. Gestión del conocimiento: Sirve como base para aplicaciones de notas modernas, permitiendo que el usuario organice ideas, tareas y proyectos con un formato consistente que no depende de un software propietario.

4) Casos de uso reales

(Da al menos 4 ejemplos concretos de dónde se usa hoy)

Ejemplos:

  • GitHub
  • Documentación técnica
  • Blogs
  • Notas personales, etc.
  • GitHub / GitLab: Los famosos archivos README.md que explican de qué trata un proyecto de software están escritos íntegramente en Markdown.
  • Obsidian / Notion: Estas herramientas de productividad utilizan Markdown para que los usuarios creen sus bases de conocimiento personales de forma rápida.
  • Stack Overflow: Millones de desarrolladores usan Markdown para dar formato a sus preguntas y respuestas en este foro técnico.
  • Generadores de Sitios Estáticos (como Hugo o Jekyll): Muchos blogs modernos y sitios de documentación oficial se escriben en archivos .md y luego se transforman automáticamente en sitios web completos.

5) Ventajas y desventajas

Ventajas

  • Legibilidad: El texto se entiende perfectamente aunque no tengas un visor especial.
  • Multiplataforma: Al ser texto plano, puedes editarlo en Windows, Mac, Linux, Android o iOS con cualquier bloc de notas.
  • Curva de aprendizaje mínima: Se aprende en menos de 10 minutos.
  • Portabilidad: Es muy fácil convertir un archivo Markdown a PDF, Word o HTML sin perder el formato.

Desventajas

  • Falta de estándar único: Existen diferentes "sabores" (flavors) de Markdown (como GitHub Flavored Markdown o CommonMark), lo que a veces causa pequeñas variaciones en cómo se ve un archivo en distintas plataformas.
  • Limitaciones de diseño: No es una herramienta de diseño gráfico; no puedes elegir fuentes, colores específicos o posiciones exactas de elementos de forma nativa.
  • Dependencia de visualizadores: Aunque es legible como texto plano, para ver el resultado final (como las imágenes renderizadas o negritas aplicadas) necesitas un editor que soporte la previsualización.
